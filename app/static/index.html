Single-patient expanded access

Individual-patient expanded-access request — Form FDA 3926

Treatment, not research (21 CFR 312.305(a)): one named patient, a serious or immediately life-threatening disease, no comparable or satisfactory alternative therapy. OSSICRO drafts the complete package the FDA reviewer receives and the request the manufacturer acts on; four independent parties own the four decisions, and OSSICRO owns none of them.

The hard line. Authorization requires four independent concurrences. OSSICRO drafts, computes clocks, and assembles — it never decides.

Input of record

Committed profile …

Before any draft is generated, a named clinician commits the confirmed intake as the input of record — an attribution of who confirmed what data these drafts are generated from. It is not a signature and executes no regulatory act: consent, IRB concurrence, causality, and submission to FDA remain their humans' non-delegable gates. The commit stores your name, the time, and SHA-256 fingerprints of each confirmed value — never the values themselves.

The profile state loads with the case — save the intake or run a check.

Statutory clocks

Deterministic working-day / calendar-day arithmetic (weekends + observed federal holidays) from physician-entered trigger dates. A clock with no trigger date stays unarmed — OSSICRO never assumes “today”.

Completeness ledger

Green = complete. Amber = drafted but awaiting a named human act (the gate shown). Red = missing or invalid. A clean concept review never promotes amber to green — only the responsible human's recorded sign-off does.

    Registry search organizer
    
      
    Find candidates — open trials & expanded-access records
    
      
    
        OSSICRO organizes a registry search for you to review — it does not recommend.
        You decide which candidates fit; confirm the basis of each against the source.
        Every candidate below is a proposal for your independent review — there is no ranking and
        no confidence score anywhere, by design.
      
    
      
    
        The search consumes only the committed profile's de-identified predicates — condition
        codes, drug, age band, sex, coded route. Names, dates, geography, identifiers, and free text
        never leave this machine. Scope: open trials and expanded-access records only; matched,
        unmatched, and unverifiable-from-structured-data criteria are shown so you can review the basis,
        never a score.
      
    
      
    
        Live registry queries are disabled in this build — results come from a mock registry adapter
        (fixture data), pending greenlight.
