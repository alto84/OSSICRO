# OSSICRO Frontend

Self-contained pharma-corporate landing page. No build step, no dependencies — open `index.html` in a browser (theme-aware light/dark).

It presents the four entry points (Patient / HCP / Micro-CRO / Pharma), the **Match → Generate → Check → Coordinate** flow, the live **completeness-ledger** example (actual `engine` output — green/amber/red), the single-patient (n-of-1) scenario and why it's lawful (312.3(b) vs 312.52), and the human-in-the-loop hard line. Links out to the GitHub repo, wiki, engine, and plan.

Design intent: the register of a pharma corporate site — restrained navy/teal palette, generous whitespace, clear IA — not a marketing splash. Next step (in `docs/frontend-strategy.md` scope): wire the four persona portals to live flows and the matching engine.
